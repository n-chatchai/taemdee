{# OAuth-return refresh + slide-up "เชื่อมต่อสำเร็จ" toast. iOS PWA standalone pushes cross-origin OAuth to Safari, so when the user finishes and returns to the PWA, the original page still shows the pre-bind state. We: 1. On connect-anchor click → save provider + timestamp in localStorage as `td_oauth_pending`. Matches both relative customer paths (/auth/{provider}/customer/start) and absolute shop URLs that bounce off the main domain (https://main.tld/auth/{provider}/start) — anchorless regex so the cross-origin shop links also flag pending. 2. On foreground (visibilitychange visible) with a fresh pending flag → move to `td_oauth_just_completed` and reload the page so the new state renders. Reload mask hides the iOS-PWA white flash behind the brand bg colour. 3. On every page load → if `td_oauth_just_completed` is set, slide up the success toast and clear the flag. Same flag works in the in-PWA OAuth case too: if iOS doesn't pop Safari, the OAuth completes within the PWA, lands on the post-OAuth page (customer: /auth/connect-complete → /my-cards; shop: /shop/dashboard) → page-load handler shows the toast based on the just-completed flag. #}
